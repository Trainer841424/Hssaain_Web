  1. 1. Domain & DNS Setup

    I started with my own domain hussain841424.in. Then I pointed its nameservers to Cloudflare and configured DNS to be managed completely from the Cloudflare dashboard.

    • Added domain to Cloudflare
    • Updated nameservers at registrar to Cloudflare NS

  2. 2. Cloudflare Tunnel on Local Server

    Next, I installed cloudflared on my local machine and authenticated it with my Cloudflare account to create a secure outbound tunnel.

    • Installed cloudflared on local server
    • Logged in via CLI and selected the domain
    • Created a tunnel pointing to my local app port (e.g. 127.0.0.1:3000)

  3. 3. Public Hostname & HTTPS

    Using Cloudflare‚Äôs dashboard, I mapped the tunnel to my hostname so any HTTPS request to the domain gets forwarded to my local service through the tunnel.

    • Created a public hostname (e.g. app.hussain841424.in)
    • Linked hostname to the tunnel
    • Let Cloudflare handle SSL/TLS and certificates

  4. 4. Apps, Subdomains & Continuous Experiments

    Finally, I created dedicated subdomains for different apps and experiments. I can push local services live quickly without worrying about router settings or public IP changes.

    Now, when I work on a new tool or mini-project, I simply link it behind the tunnel and expose it via a neat subdomain under my own domain.
